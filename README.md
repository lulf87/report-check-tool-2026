# 报告核对工具

本项目用于上传 PDF 检验报告、PTR 产品技术要求或检测原始记录，抽取 PDF 文本和版面证据后进行自动核对；需要图文判断或复杂语义判断时会调用 Codex CLI 辅助复核。

## 功能

- 报告自身核对：检查报告内部字段、页码、结论、表格结果等一致性。
- PTR 与报告核对：按报告首页“检验项目”识别 PTR 第 2 章性能指标范围，核对报告“标准要求”是否完整摘录。
- 原始记录与报告核对：对照原始记录和检验报告中的标准条款、检验要求、判定和实测证据，支持 `GB 9706.1-2020` 和 `GB 9706.202-2021`。
- 核对结果会按错误、需复核、通过分组展示证据明细，并支持从结果页导出为 PDF。

## 核对模式

### 报告自身核对

用途：只上传检验报告 PDF，检查报告内部的一致性问题，例如首页信息、页码、检验结论、表格结果和备注之间是否冲突。

使用方式：

1. 在页面选择“报告自身核对”。
2. 上传检验报告 PDF。
3. 点击“开始核对”，等待进度完成。
4. 在结果区优先查看“错误”和“需复核”项目，必要时导出 PDF 归档。

### PTR 与报告核对

用途：同时上传 PTR 产品技术要求 PDF 和检验报告 PDF，核对报告是否按 PTR 中声明的性能指标和条款完整摘录标准要求。

使用方式：

1. 在页面选择“PTR 与报告核对”。
2. 分别上传 PTR PDF 和对应检验报告 PDF。
3. 点击“开始 PTR 核对”。
4. 查看首页声明条款、报告实际条款、缺漏条款，以及最细条款的“PTR 内容 / 报告内容”对照。

### 原始记录与报告核对

用途：同时上传检测原始记录 PDF 和检验报告 PDF，核对报告中的标准条款、判定和实测证据是否能在原始记录中找到对应依据。

使用方式：

1. 在页面选择“原始记录与报告核对”。
2. 分别上传原始记录 PDF 和对应检验报告 PDF。
3. 在“标准”中选择 `GB 9706.1-2020` 或 `GB 9706.202-2021`。
4. 选择执行模式：
   - `快速核对（推荐）`：程序能确定一致或不一致的项目会直接给出结果；证据缺失、判定不确定或需要图文判断的项目再交给 Codex。
   - `完整 Codex`：每个核对序号都会提交 Codex 复核，适合对结果进行更保守的二次检查，但耗时更长。
5. 根据需要调整并发数。并发越高速度越快，但会同时发起更多 Codex 任务。
6. 点击“开始原始记录核对”，完成后先处理“错误”和“需复核”项目。

## 目录

- `backend/`：FastAPI 后端，负责 PDF 解析、证据构建和 Codex 调用。
- `frontend/`：Vite + React 前端。
- `docs/`：需求和实现计划文档。

`素材/`、`tmp/`、`frontend/node_modules/`、`frontend/dist/` 等目录不会提交到 GitHub。

## 注意

- 本仓库不提交原始 PDF 素材；需要核对时从前端页面重新上传 PDF。
- 需要本机已登录并可使用 Codex CLI，否则后端会返回 Codex 调用或解析相关诊断。
- `快速核对` 的通过项表示程序已根据可抽取证据完成一致性判断；`需复核` 表示证据不足、映射不确定或图文判断置信度不够，不应静默视为通过。
